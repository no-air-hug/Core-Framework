# Pickup Point Delivery Option Generator

A Shopify Function that provides Click & Collect pickup points powered by OneStock. It uses the `purchase.pickup-point-delivery-option-generator` function type with two stages: `fetch` (builds a request to the proxy) and `run` (maps the response into Shopify pickup points).

## How It Works

```
Shopify Checkout (customer enters address)
     |
     v
fetch() — reads lat/lon, cart, shop metafields
     |  builds POST request to proxy
     v
Hydrogen Proxy on Oxygen
     |  POST /api/onestock/pickup-points
     |  → calls OneStock /endpoints (nearby stores)
     |  → calls OneStock /delivery_promises per store
     v
run() — receives HTTP response
     |  maps delivery points to Shopify PickupPointDeliveryOption
     |  attaches onestock.delivery-promise metafield to each point
     v
Checkout displays pickup points in "Pick up" tab
```

### fetch stage (`src/fetch.ts`)

1. Validates the delivery address has lat/lon coordinates
2. Reads `integrations.onestock_delivery_mapping` shop metafield to get C&C method IDs
3. Reads `integrations.onestock_proxy_url` for the proxy URL
4. Builds a POST request to `{proxyUrl}/api/onestock/pickup-points` with lat, lon, and cart data
5. Sets a 2 second read timeout

### run stage (`src/run.ts`)

1. Parses the proxy response (`{deliveryPoints: [...]}`)
2. Maps each delivery point to a Shopify `PickupPointDeliveryOption` with:
   - Address, coordinates, business hours
   - Provider name (using the collect promise text from OneStock)
   - A `onestock.delivery-promise` metafield containing the delivery promise data
3. The existing `delivery-promise.tsx` UI extension watches for pickup point selection and updates the cart metafield

## Key Differences from Reference Project

- No `delivery.onestock_available_for_collection` product metafield check — OneStock determines eligibility via the delivery promise response status
- No brand-specific provider logos — uses a generic provider name
- No membership product filter

## Shop Metafields Required

These are the same metafields used by the delivery promise extensions:

| Namespace | Key | Used For |
|-----------|-----|----------|
| `integrations` | `onestock_proxy_url` | Proxy URL for the fetch request |
| `integrations` | `onestock_sales_channel` | OneStock sales channel ID |
| `integrations` | `onestock_delivery_mapping` | JSON mapping of display names to OneStock method IDs |

The delivery mapping must include a `"Click and Collect"` key, e.g.:
```json
{"Standard Delivery":"standard_delivery","Express Shipping":"express_shipping","Click and Collect":"c&c_standard"}
```

## Setup

### 1. Install Dependencies

```bash
cd apps/onestock-extensions/extensions/pickup-point-delivery-option-generators
pnpm install
```

### 2. Generate Types

The `schema.graphql` was copied from the reference project (same function type). To generate the TypeScript types, run codegen directly from the extension directory:

```bash
cd apps/onestock-extensions/extensions/pickup-point-delivery-option-generators
npx graphql-codegen --config package.json
```

**Note:** Do NOT use `pnpm shopify app function typegen` — Shopify CLI internally uses `npm exec` which resolves to an ancient deprecated `graphql-code-generator` package instead of the local `@graphql-codegen/cli`. Running `npx graphql-codegen` directly uses the correct local binary.

If you modify the GraphQL queries in `src/fetch.graphql` or `src/run.graphql`, re-run the codegen command above to regenerate `generated/api.ts`.

### 3. Deploy

```bash
pnpm shopify app deploy
```

### 4. Activate the Function

Run via GraphiQL (connected to the OneStock Delivery Promise app):

```graphql
mutation {
  pickupPointDeliveryOptionGeneratorActivate(
    functionId: "YOUR_FUNCTION_ID"
  ) {
    userErrors {
      field
      message
    }
  }
}
```

Get the function ID from:
```graphql
{
  shopifyFunctions(first: 25) {
    nodes {
      id
      title
      apiType
    }
  }
}
```
